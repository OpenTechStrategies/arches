Arches 8.0.3 Release Notes
--------------------------

### Bug Fixes and Enhancements

- Fix migration failing to create missing node aliases [#12359](https://github.com/archesproject/arches/pull/12359)
- Promote lifecycle state management from `Resource` to `ResourceInstance` to better support subclassing [#12321](https://github.com/archesproject/arches/issues/12321)
- Adds the `resource_instance_lifecycle_state_id` column to the `__arches_instance_view_update()` function [#12336](https://github.com/archesproject/arches/issues/12336)
- Generate `featureid` on update/insert of geometries in relational models [#12343](https://github.com/archesproject/arches/pull/12343)
- Fix error when logged out user visits edit history [#12386](https://github.com/archesproject/arches/issues/12386)
- Fixes bug in tile excel importer that interpreted all excel files as containing arches business data [#12364](https://github.com/archesproject/arches/issues/12364)
- Remove no-op `get_interchange_value()` datatype method [#12324](https://github.com/archesproject/arches/pull/12324)
- Fix Advanced Search for `resource-instance` datatypes [#12334](https://github.com/archesproject/arches/pull/12334)
- Fix error where map popup template renders as string not template content [#12339](https://github.com/archesproject/arches/pull/12339)

### Upgrading Arches

1. Upgrade to version 8.0.0 before proceeding by following the upgrade process in the [Version 8.0.0 release notes](https://github.com/archesproject/arches/blob/dev/8.0.x/releases/8.0.0.md)

2. Upgrade to Arches 8.0.3
    ```
    pip install --upgrade arches==8.0.3
    ```

3. Add `"django_sql_migrate"` to `INSTALLED_APPS`.

4. Run `python manage.py migrate`.

5. If you are running Arches on Apache, restart your server:
    ```
    sudo service apache2 reload
    ```
